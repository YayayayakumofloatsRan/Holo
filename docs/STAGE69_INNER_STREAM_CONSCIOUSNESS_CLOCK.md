# Stage69 Inner Stream Consciousness Clock

Stage69 adds a bounded, always-on internal consciousness stream to the WSL subject kernel.

## Goal

Holo now has an endogenous micro-tick that advances even when no external message arrives and calls the configured LLM processor for a compact internal micro-thought. The stream is recurrent: each tick updates a small internal dynamic field, and the next processor call sees that field again. This is the first runtime substrate for measuring continuous subject-state flow instead of only turn-bound prompt traces or low-frequency reflection jobs.

## Runtime Shape

The new loop is `inner_stream`.

Each tick records:

- `sensory_edge`
- `attention_focus`
- `affective_tension`
- `memory_echo`
- `goal_pressure`
- `inhibition_gate`
- `candidate_action`
- `field_state`
- `plasticity_trace`
- `recurrent_context`

The tick is generated by `holo_host/inner_stream.py` and is driven from `HoloDaemon.run_cycle()`. It uses local runtime state, latest brain-loop telemetry, idle time, and a bounded `inner_stream_thought` processor call routed through the existing processor fabric. By default this uses `subject_main`, so DeepSeek deployments use the configured pro/main DeepSeek model for the consciousness-flow micro-thought, with `micro_fast` available as fallback.

`field_state` is the biomimetic substrate: activation energy, prediction error, salience, affective tension, dominant attractor, and recurrence depth. `plasticity_trace` carries recent micro-thoughts, attractor counts, and the last memory echo. The LLM sees both in the next prompt, so its next thought is conditioned by the previous inner dynamics rather than being an isolated telemetry sample.

## Authority Boundary

The stream is deliberately bounded:

- memory write: `volatile_ring_only`
- self-memory write: false
- policy write: false
- transport write: false
- model call: true when `inner_stream_model_enabled=true`

This means Stage69 adds a continuous model-backed internal activity surface without adding a second decision layer or a transport-side actor. The model call is processing, not external authority. Inhibition, volatile scope, and delayed consolidation are treated as part of the biological control process: long-term memory sedimentation still has to pass through the existing maintenance, reflection, and consolidation gates.

## Observability

`brain_status()` now exposes:

- `inner_stream` as a brain loop
- `inner_stream_state` with latest tick and recent volatile ticks

The loop also writes normal `brain_loop_runs` telemetry, so status views and later visualization tools can sample it alongside heartbeat, attention, homeostasis, affect, and higher cognitive loops.

## Stability

Brain-loop runner exceptions are now recorded as `status=error` loop telemetry instead of crashing the daemon cycle. This is required because a continuous internal stream must never become a single point of kernel failure.
