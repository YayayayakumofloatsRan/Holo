# Stage88 Within-Thread Self-Organization

## Purpose

Stage88 implements the next biomimetic requirement after Stage87:

```text
Outcome-conditioned local adaptation over the current transcript.
```

The point is self-organization inside the active thread, not a claim of
persistent self-memory. A useful biomimetic system should change the next
working field after observing the last interaction outcome: what the user asked,
what input was missing, which answer form worked, and which self-learning or
memory claim had to be blocked.

## Mechanism

Stage88 adds a simulation-local adaptation signal to the Stage42 user-sim
memory path:

- `stage88.stage = stage88-within-thread-self-organization`
- `scope = current_thread_only`
- `learning_signal = current-thread evidence update from N observed turns`
- `missing_input_targets`
- `blocked_claims`
- `useful_response_forms`
- `next_turn_instruction`

The signal is generated by `_IsolatedNoviceMemory._local_adaptation()` and is
included in the sidecar packet as `stage88`. The bionic kernel exposes it in
`capsule.working_field.local_adaptation`, and the provider prompt receives it
as `Stage88 current-thread adaptation`.

The active-inference interpretation is bounded:

```text
last interaction outcome -> local evidence update -> next attention target
```

This is closer to self-organization than Stage87 because the second turn is
not only a fresh prompt with continuity text. It receives an explicit local
adaptation record derived from the previous turn.

## Provider Guards

Stage88 also tightens two provider failure modes found during live DeepSeek
cells:

- generic assistant identity is rewritten into `current-thread interaction
  partner`
- unverified cross-conversation memory/self-learning claims preserve
  continuity instead of collapsing into a generic safety line

The memory guard is query-specific:

- continuity probes preserve what the thread was covering
- summary probes preserve image-input limits and practical capability
- biomimetic probes answer with working-memory, attention, relevant-turn, and
  current-query language

This keeps the result biomimetic rather than merely safe.

## Evidence

Red tests first failed because:

- no `local_adaptation` existed in the second-turn working field
- provider prompts did not receive current-thread adaptation
- assistant-shell identity was not rewritten
- memory-overclaim guard lost continuity on "what were we talking about"
- summary and brain-like prompts fell back to the same generic guard text

Implemented tests:

```powershell
python -m pytest tests\test_stage42_bionic_user_sim.py -q
```

Result:

```text
19 passed
```

Offline 20-turn free dialogue:

```text
overall_score=0.8981
interaction_usefulness_score=0.793
continuity_score=0.8667
issue_count=0
second_adaptation.stage=stage88-within-thread-self-organization
second_adaptation.scope=current_thread_only
second_adaptation.missing_input_targets=["one concrete task or current facts"]
```

```text

```

This is the key Stage88 improvement over Stage87 provider novice, where
continuity had remained weak at `0.5333`.

DeepSeek V4 Pro free dialogue, 8 turns:

```text
overall_score=0.9787
interaction_usefulness_score=0.885
continuity_score=1.0

duplicate_followup=false
```

A 12-turn provider run exceeded the local command timeout and is not counted as
acceptance evidence. An earlier 12-turn run before the query-specific guard
repair failed on duplicate follow-up despite good usefulness and continuity;
that failure motivated the query-specific memory guard.

## Interpretation

Supported:

- current-thread adaptation is visible in the working field
- provider prompts receive the adaptation signal
- provider novice continuity improved from Stage87's `0.5333` to `1.0`
- short provider free dialogue passed with no low-usefulness or duplicate
  follow-up issue

Not supported:

- no persistent autobiographical self-memory
- no human consciousness claim
- no claim that long-run provider dialogue is solved; 12-turn evidence still
  needs a stable repeated campaign

## Next Gate

Stage89 should make this self-organization less heuristic:

```text
learned local policy over transcript outcome labels.
```

The next mechanism should turn the Stage88 adaptation record into a small
online policy vector: ask-for-specific-task, preserve-continuity, answer-
biomimetic-structure, visual-boundary, and reduce-repetition. Acceptance should
require repeated provider cells, not only one 8-turn success.
