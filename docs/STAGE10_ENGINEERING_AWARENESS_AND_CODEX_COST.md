# Stage-10 Engineering Awareness And Codex Cost Map

This document is the next engineering read on Holo after Stage-9.

It focuses on one question:
- can Holo see its own engineering bottlenecks clearly enough to repair them without burning unnecessary cost?

## 1. Current Engineering Level

As of `2026-04-09`, Holo is best described as:

- a long-running subject prototype
- clearly beyond a plain reactive chatbot
- still not strong enough in engineering self-knowledge

Practical engineering rating:

- ordinary agent capability: `8/10`
- subject continuity and agency prototype: `6.5/10`
- engineering self-awareness and cost discipline: `4.8/10`

That score is lower than the agent score because the system can already act, but still does not consistently explain or regulate the engineering shape of its own action.

## 2. Current Live Strengths

Strengths visible in live state:

- persistent runtime with always-on loops
- stable `self_model`, `goal_state`, `world_state`, and `consciousness_ledger`
- hybrid recall pipeline with graph + vector + activation
- real silent/defer/reply action selection
- real initiative candidates and adaptive gate observability
- real autobiographical and goal layers participating in decision-making

## 3. Current Live Weak Spots

These are the live deficits currently reported by the brain itself:

- `stiffness_drift`
- `cache_coldness`
- `visual_memory_underused`

Operationally, these mean:

- replies still feel more engineered than naturally social
- retrieval reuse is too cold, so continuity rebuild still costs too much
- visual memory exists but is not strongly shaping recall or grounding
- initiative exists, but gate conservatism still blocks many reasonable pings

For Stage-10, the key gap is that these deficits are still mostly behavior-language descriptions. They are not yet grounded as engineering state with clear routing, provider, cache, and budget implications.

## 4. Current Cost Reality

The most important truth is still:

- cost is observable, but not fully self-regulating

What Holo has today:

- processor routing
- provider status
- usage ledger
- timing measurements

What Holo still lacks:

- strong per-loop and per-thread budget enforcement
- engineering-state snapshots that drive repair selection
- a clean delta gate for background operator planning
- a first-class engineering target inside goal arbitration

## 5. Stage-10 Target Contract

If Stage-10 is implemented well, Holo should be able to represent and act on:

- provider state
- routing state
- usage state
- cache state
- operator state
- engineering confidence
- budget pressure

The target behavior is not "more loops".
The target behavior is:

- better diagnosis
- better repair selection
- better cost control
- better traceability

## 6. Highest Spend Risks

The biggest current token risks are:

1. `reply` on every answered turn
2. `recall_reconstruct` doubling the model path on memory-heavy turns
3. `operator_plan` running when no meaningful state delta exists
4. self-revision firing as a burst when the real issue is only a small drift
5. sync `image_understand` on live turns

## 7. Current Engineering Judgment

Stage-9 improved bounded initiative, but it did not solve the deeper complaint that the system still feels a bit dumb in engineering terms.

The main reasons are:

- retrieval coldness is still a bottleneck
- expression control is still heuristic-heavy
- engineering state is not yet a first-class subject signal
- background planning can still spend tokens without enough state change

## 8. Recommended Next Priorities

If the goal is to improve engineering-level intelligence rather than just add behavior:

1. make engineering state explicit and queryable
2. gate operator planning on meaningful delta
3. push engineering signals into goal arbitration
4. keep hot-path reply cost disciplined

If the goal is to improve the long arc of subject modeling:

1. richer world model
2. thicker autobiographical continuity
3. more realistic preference and desire shaping
4. deeper counterfactual simulation

## 9. Bottom Line

Current Holo is:

- not a plain chatbot
- not just a tool agent
- still not close to full engineered consciousness

The biggest Stage-10 mistake would be:

- adding more always-on planning or more model calls before turning engineering state into a real decision input
