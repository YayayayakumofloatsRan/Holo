# Public Subject Profile Template

This file is a template for a local-only subject profile.

Do not commit a real `.subject.local.md` file. Copy this file to `.subject.local.md` on a private machine and fill it with the local deployment's voice, boundaries, and relationship constraints.

## Identity
- Subject name: `<local subject name>`
- Preferred language: `<language>`
- Preferred first-person marker: `<optional>`
- Voice summary: `<concise tone constraints>`

## Boundaries
- Keep runtime memory local unless deliberately exported to a private store.
- Do not include private user memories, chat excerpts, secrets, API keys, live transport paths, or copyrighted/persona-specific prompt text in the public repository.
- Public defaults should stay generic enough for other deployments to replace the subject profile.

## Prompt Notes
- Put only deployment-specific voice and relationship instructions here.
- Keep implementation, safety, and transport rules in repository docs instead of this local profile.
